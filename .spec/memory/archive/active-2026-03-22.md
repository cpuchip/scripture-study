## Architecture Quick Reference

### brain-app key files
| File | Purpose |
|------|---------|
| `lib/main.dart` | App entry + widget background callbacks + all secondary entrypoints |
| `lib/screens/today_screen.dart` | Main Today tab |
| `lib/services/becoming_api.dart` | REST client for ibeco.me |
| `lib/services/widget_service.dart` | Push data to Android widget |
| `lib/widgets/practice_form.dart` | Shared practice creation form |
| `android/.../PracticeWidgetProvider.kt` | Practice widget provider |
| `android/.../PracticeWidgetService.kt` | RemoteViews factory |

### ibeco.me key files
| File | Purpose |
|------|---------|
| `cmd/server/main.go` | HTTP routes, middleware, server entry |
| `internal/schedule/schedule.go` | Practice scheduling logic |
| `internal/db/` | SQLite + PostgreSQL DB layer |
| `internal/notify/` | Web Push scheduler + HTTP handlers |
| `frontend/` | Vue 3 + Tailwind web UI |

### Widget background callback flow
1. Kotlin widget button tap → PendingIntent with URI
2. HomeWidgetBackgroundReceiver → Dart backgroundCallback() in main.dart
3. Dart parses URI, optimistic update, API call, widget update
4. Kotlin provider re-renders
